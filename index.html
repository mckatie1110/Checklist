Web Design Checklist

HTML Rules

  • Do not label classes with a tag or html element.
  • Keep your code neat by using indentions and proper spacing in between elements.
  • If you have a nav in your site format it using an unordered list with list elements for the tabs.

CSS Rules

  • Code from top to bottom or by element to keep everything organized, remember that styles "cascade" down to all other elements unless you change them.
  • Keep your code neat by spacing in between elements.
  • Always change the defaults - you are a designer!

Technical and File Management Rules

  • Keep your code organized as you go.
  • Style your css by grouping all like items together or organizing by pages.

Content

  • Body text should only be 45-70 characters long per line.
  • Organize content in order of importance -heirarchy

Design Foundations

Grid

  • ALWAYS use a grid
  • Make the grid on it's own layer and lock it to prevent accidently adjusting the gridlines.
  • Good starting grid is 12x12.

Typography

  • Do not use more than two font-familes per page.
  • Do not use Fake smallcaps, Fake bold, or Fake anything. Find a font-family with built in styles.
  • Change the line spacing to at least 1.5 time more what the font size is to allow for easier reading.

Color Theory

  • Check colors with the color blindness tests for high contrast.
  • Label colors with hexcodes so they can easily be copied and looked up.

Using Style Tiles

  • Label everything! Colors with hexcodes. Fonts with font styles and sizes.
  • Use whitespace to put space in between each design element.

Saving Images Correctly

  • Always use JPEG or JPG for photography.
  • GIF is typically the smallest size.
  • Never use PNG-8 use PNG-24 instead.
  • JPEG GIF and PNG are Not Scalable. They will get pixilated.

Accessibility/Usability

  • Check with blindness test to validate usability.
  • Add alternate fonts and colors so that if the users computer does not have the font/color you programmed, the browser loads the alternative.
  • Add text alternatives for images that will appear if the image does not load or if the user is using a text to speech device.

Code Validation

  • Always check code validation on HTML and CSS.
  • Make corrections accordingly.

Git/Terminal

  • NEVER "git init" in your Home Directory.
  • Add, Commit, Push!
  • Always add a comment to the commit command. Make your comments descriptive

SASS

  • The command for sass watch is: sass --watch "style.scss"
  • Run sass watch from the project folder... do not have your scss file in a "spreadsheets" folder
